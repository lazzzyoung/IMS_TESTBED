# VoLTE Mutation Fuzzer

> [!info] 한 줄 요약
> VoLTE/IMS 단말의 SIP 스택 견고성을 검증하기 위한 **터미널 기반 SIP 뮤테이션 퍼저**.

## 목적

VoLTE/IMS 환경에서 동작하는 모바일 단말(UE)의 SIP 스택에 **비정상 SIP 패킷**을 전송하여 크래시, ANR, 재부팅 등 이상 동작을 탐지하는 **블랙박스 퍼징 프레임워크**이다.

> [!important] 핵심 가치
> - IANA 기준 **14개 요청 메서드 + 75개 응답 코드** 전수 커버
> - **3계층 뮤테이션** (모델 / 와이어 / 바이트)으로 다양한 결함 탐색
> - 시드 기반 **재현 가능한** 뮤테이션
> - 소프트폰(Baresip) 및 실제 UE 직접 전송 지원

## 아키텍처

### 파이프라인 개요

```
┌────────────┐    ┌────────────┐    ┌──────────────────┐    ┌────────────┐
│  Generator │ →  │  Mutator   │ →  │ Sender / Reactor │ →  │   Oracle   │
│            │    │            │    │                  │    │            │
│ 정상 SIP   │    │ 뮤테이션    │    │ 소프트폰 / 실UE   │    │ 크래시 탐지  │
│ 패킷 생성   │    │ 적용       │    │ 전송 + 응답 수집   │    │            │
└────────────┘    └────────────┘    └──────────────────┘    └────────────┘
                                                                  │
                                                            ┌────────────┐
                                                            │  Campaign  │
                                                            │ 자동화 루프  │
                                                            └────────────┘
```

> [!note] 현재 구현 상태
> Phase 1~5 구현 완료. 상세 구현 내용은 [[구현-문서]] 참고.

### 모듈 구조

```
src/volte_mutation_fuzzer/
├── sip/              # SIP 프로토콜 모델 (카탈로그, 공통 타입, 렌더링)
├── generator/        # 정상 베이스라인 패킷 생성
├── mutator/          # 뮤테이션 엔진 (3계층)
├── sender/           # 패킷 전송 및 응답 수집
├── oracle/           # 이상 판정 (소켓 + 프로세스 + 로그)
├── campaign/         # 캠페인 자동화 (케이스 생성, 실행, 결과 저장)
├── softphone.py      # Baresip 런처
└── packet_docs.py    # 패킷 문서 생성 유틸
```

## 핵심 모듈 요약

| 모듈 | 역할 | 핵심 클래스 |
|------|------|-----------|
| `sip/` | SIP 프로토콜 모델 (14 메서드 + 75 응답코드 카탈로그) | `SIPCatalog`, `SIPURI`, `ViaHeader` |
| `generator/` | RFC 기반 정상 패킷 생성 | `SIPGenerator`, `GeneratorSettings`, `DialogContext` |
| `mutator/` | 3계층 뮤테이션 (model/wire/byte), seed 재현 | `SIPMutator`, `EditableSIPMessage`, `MutatedCase` |
| `sender/` | UDP/TCP 전송, 응답 수집. softphone + real-ue-direct | `SIPSenderReactor`, `SendReceiveResult` |
| `oracle/` | 소켓 + 프로세스 + 로그 3축 이상 판정 | `SocketOracle`, `ProcessOracle`, `LogOracle`, `OracleEngine` |
| `campaign/` | Tier 기반 자동 캠페인, JSONL 저장, report/replay | `CaseGenerator`, `CampaignExecutor`, `ResultStore` |

> [!tip] 상세 구현
> 각 모듈의 설계 배경, 동작 방식, 코드 구조는 [[구현-문서]] 참고.

## CLI 사용법

### 진입점

```bash
uv run fuzzer <subcommand>
```

### 패킷 생성

```bash
# 요청 생성
uv run fuzzer request OPTIONS
uv run fuzzer request INVITE --override '{"Max-Forwards": 10}'

# 응답 생성
uv run fuzzer response 200 INVITE \
  --context '{"call_id":"c1","local_tag":"tag","local_cseq":7}'
```

### 뮤테이션

```bash
# 직접 생성 + 뮤테이션
uv run fuzzer mutate request OPTIONS --strategy state_breaker --layer model

# 파이프라인 뮤테이션
uv run fuzzer request OPTIONS | \
  uv run fuzzer mutate packet --strategy header_chaos --layer wire
```

### 전송

```bash
# 소프트폰 대상
uv run fuzzer send request OPTIONS \
  --target-host 127.0.0.1 --target-port 5060

# 실 UE 대상
uv run fuzzer send packet \
  --mode real-ue-direct --target-msisdn 222222

# 전체 파이프라인
uv run fuzzer mutate request INVITE --layer byte | \
  uv run fuzzer send packet --target-host 10.20.20.2
```

### 캠페인

```bash
# 캠페인 실행
uv run fuzzer campaign run \
  --target-host 10.20.20.2 --scope tier1 --max-cases 1000 \
  --process-name baresip --log-path /var/log/baresip.log

# 결과 분석
uv run fuzzer campaign report results/campaign.jsonl --filter suspicious,crash

# 특정 케이스 재현
uv run fuzzer campaign replay results/campaign.jsonl --case-id 42
```

## 설계 결정

> [!abstract] 핵심 설계 원칙

1. **Pydantic 기반 검증** — 모든 SIP 구조체에 런타임 검증 + JSON 직렬화 적용
2. **3계층 뮤테이션** — 시맨틱(model) / 구조(wire) / 비트(byte) 분리로 유연한 결함 탐색
3. **카탈로그 주도** — 14개 요청 + 75개 응답을 중앙 카탈로그에서 일관 관리
4. **환경변수 기반 설정** — Docker/실험실 배포 지원
5. **소프트폰 우선 전략** — 로컬 Baresip 검증 후 실 UE 전송으로 단계적 확장
6. **동기 I/O** — 단순성 우선, 비동기는 추후 확장 가능
7. **파이프라인 조합** — Unix 철학에 따라 `generate | mutate | send` 파이프 체인 지원
