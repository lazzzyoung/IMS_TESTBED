# 2026-05-02 — UE1/UE2 SIP MESSAGE 송수신 트러블슈팅 체크리스트

## 현재 상태 요약

- LTE attach: 성공
- IMS REGISTER: 성공
  - `401 -> 200` 패턴 확인
- INVITE baseline: 별도 성공 기록 있음
- `MESSAGE` 1건 송신 시도: 현재까지 모두 `timeout`
- Wireshark: `MESSAGE` 미관측
- UE 앱 반응: 없음

## 가장 중요한 현재 결론

- 현재 `campaign run --mode real-ue-direct --mt --methods MESSAGE` 경로는
  `P-CSCF/S-CSCF`를 통과하는 일반 SIP 프록시 경로가 아니라,
  `pcscf` 컨테이너 네트워크 네임스페이스에서 **UE contact IP:port로 직접 송신**하는 경로다.
- 따라서 `scscf/pcscf` 로그에 `MESSAGE`가 안 보인다고 해서 반드시
  "패킷이 안 만들어졌다"라고 단정할 수는 없다.
- 대신 현재는 **직접 송신 경로에서 응답을 못 받고 timeout 나는 상태**로 보는 것이 맞다.

근거 코드:

- `ipsec_mode == "null"` / `"bypass"` 이면 `bind_container = "pcscf"` 설정
- 실제 송신은 `send_via_container(...)` 로 `pcscf` 컨테이너 안에서 수행
- target 은 `resolved_host:resolved_port` 즉 UE contact 로 직접 향함

관련 파일:

- `src/volte_mutation_fuzzer/campaign/contracts.py`
- `src/volte_mutation_fuzzer/campaign/core.py`
- `src/volte_mutation_fuzzer/sender/core.py`

### 6. `MESSAGE`가 Kamailio 로그에 안 보임

- 증상:
  - `docker logs pcscf/scscf` 에 `MESSAGE` 없음
- 현재 해석:
  - 이번 테스트 경로는 Kamailio 프록시 통과 경로가 아니라
    `pcscf` 컨테이너에서 UE contact 로 직접 보내는 경로일 가능성이 큼
- 상태:
  - [x] 확인
  - [x] 코드 경로로 근거 확보

### 7. 가장 유력한 현재 원인: UDP/TCP transport mismatch

- 관측 근거:
  - `pcscf` NAT ping 로그는 반복적으로
    - `OPTIONS to sip:001010000123528@10.20.20.16:8200 ... transport=tcp`
    - `OPTIONS to sip:001010000123542@10.20.20.15:31800 ... transport=tcp`
  - 즉, 현재 live contact 검사는 TCP 기준으로 성공 중
  - 반면 `campaign run` 기본 transport 는 `UDP`
- 의미:
  - 현재 direct MESSAGE 는 **TCP contact 에 UDP 로 쏘고 있을 가능성**이 큼
  - 이 경우 조용히 timeout 날 수 있음
- 상태:
  - [x] 강한 의심 근거 확보
  - [ ] 아직 `--transport TCP` 재시도 안 함

### 8. 현재 Wireshark 위치/필터가 direct path 와 안 맞을 수 있음

- 증상:
  - Wireshark 에 아무것도 안 보임
- 현재 해석:
  - direct path 가 `pcscf` 컨테이너 내부에서 UE IP 로 바로 나가므로
    host 의 잘못된 인터페이스/필터에서는 안 보일 수 있음
- 상태:
  - [x] 가능성 확인
  - [ ] `br-volte` 또는 `pcscf` 컨테이너 내부 tcpdump 로 재확인 필요

## 다음 우선순위 체크리스트

### A. transport mismatch 확인

- [ ] 아래 명령으로 `TCP` 전송 재시도

```bash
uv run fuzzer campaign run \
  --mode real-ue-direct \
  --methods MESSAGE \
  --transport TCP \
  --target-msisdn 222222 \
  --impi 001010000123528 \
  --from-msisdn 111111 \
  --mo-contact-host 10.20.20.16 \
  --mo-contact-port-pc 8200 \
  --mo-contact-port-ps 8201 \
  --layer wire \
  --strategy identity \
  --max-cases 1 \
  --timeout 10 \
  --no-process-check \
  --no-adb \
  --no-pcap \
  --mt
```
