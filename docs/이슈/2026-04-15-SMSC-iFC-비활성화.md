# 2026-04-15 — SMSC iFC 비활성화로 MESSAGE 478 Unresolvable Destination 해결

## 증상

`--methods MESSAGE --mode real-ue-direct` 퍼징 실행 시 S-CSCF(172.22.0.20)가 P-CSCF(172.22.0.21)에 **478 Unresolvable Destination** 반환. 패킷은 단말(A31)까지 도달하지 못함.

재현 커맨드:
```bash
uv run fuzzer campaign run --mode real-ue-direct --target-msisdn 111111 \
  --mt --ipsec-mode bypass --methods MESSAGE --layer wire \
  --strategy identity --timeout 6 --max-cases 1
```

## 근본 원인

S-CSCF가 PyHSS iFC 규칙에 따라 MESSAGE를 SMSC AS로 포워딩 시도하지만, SMSC 컨테이너가 배포되지 않아 호스트 네임 해석 실패.

`docker logs scscf` 핵심 라인:
```
ims_isc: MATCH -> sip:smsc.ims.mnc001.mcc001.3gppnetwork.org:5060
<core> [core/resolve.c:1777]: sip_hostport2su(): could not resolve hostname
tm [ut.h:302]: uri2dst2(): failed to resolve
tm [t_fwd.c:1771]: t_forward_nonack(): failure to add branches
```

흐름: Fuzzer → P-CSCF → **S-CSCF (iFC 매치 → SMSC 포워딩 실패)** → 478 → P-CSCF → Fuzzer. 단말 브랜치는 생성 자체가 안 됨.

## 해결

PyHSS의 iFC는 API가 아닌 **Jinja2 XML 템플릿**으로 관리됨 (`infrastructure/pyhss/default_ifc.xml`, 컨테이너 `/mnt/pyhss/default_ifc.xml`로 bind mount).

`<!-- SIP MESSAGE Traffic -->` InitialFilterCriteria 블록을 주석 처리해 SMSC 포워딩 트리거 제거. 결과적으로 S-CSCF는 iFC 체크 통과 후 표준 IMS 라우팅으로 터미네이팅 측으로 바로 전달.

### 퍼징 관점 효과
- 패킷이 실제 A31 SIP 파서까지 도달 → MESSAGE/RP-DATA 변이가 UE 파서를 실제로 공격
- stack_failure/crash verdict 의미 있어짐
- 주의: SMS 시맨틱은 깨짐 (SMSC가 RP-DATA를 ack하지 않음). 실제 SMS 수신이 아닌 SIP MESSAGE 전달만 수행됨.

## 재현 가능성 (poe 태스크 통합)

`scripts/provision_subscribers.py`에 `apply_pyhss_ifc_template()` 추가:
- 저장소 XML과 컨테이너 bind mount 내용 비교 (불일치 시 경고)
- `docker restart pyhss` 수행해 iFC 캐시 무효화

→ `poe provision` 한 번 실행으로 가입자 upsert + iFC 템플릿 반영 + pyhss 재시작이 원샷으로 이뤄짐.

### 서버에서 실행
```bash
cd /home/ubuntu/Desktop/fuzzer && poe provision
```

### 필수 후속 작업
- **A31 재등록**: 비행기모드 on/off. 새 iFC는 다음 MAR(Multimedia Auth Request) 시점에 렌더링되므로 재REGISTER 필요.

## 변경 파일
- `infrastructure/pyhss/default_ifc.xml`: MESSAGE→SMSC InitialFilterCriteria 블록 XML 주석으로 비활성화
- `scripts/provision_subscribers.py`: `apply_pyhss_ifc_template()` 추가, `main()`에서 `provision_pyhss()` 이후 호출

## 복구 (SMSC AS 배포 시)

`default_ifc.xml`의 주석 해제 후 `poe provision` 재실행. SMSC 컨테이너가 `smsc.ims.mnc001.mcc001.3gppnetwork.org:5060`로 도달 가능해야 함 (DNS 또는 `/etc/hosts` 등록).
