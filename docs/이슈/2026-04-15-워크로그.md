## 2. MESSAGE 기본 payload를 seed 기반으로 개선

### 변경 이유

기존 `build_mt_packet()` 경로에서:

- `INVITE`만 동적인 SDP body가 있었고
- `MESSAGE`는 항상 `"test"`
- 나머지 메서드는 대부분 빈 body

여기서 `MESSAGE`는 mutator가 개입하더라도 **기본 body가 너무 단순해서 시드별 다양성이 약했다.**

### 변경 내용

파일:

- `src/volte_mutation_fuzzer/generator/mt_packet.py`

추가:

- `_MESSAGE_BODY_ALPHABET`
- `_build_default_message_body(seed: int)`

동작:

- 같은 `seed` → 같은 body
- 다른 `seed` → 다른 body
- explicit `body=` 전달 시 기존처럼 override

예시 형식:

```text
VMF MESSAGE seed=<seed> payload=<deterministic token>
```

이로써 재현 가능성을 유지하면서도 `"test"` 고정값 문제를 제거했다.

## 3. MT non-INVITE body/header 생성 확장

### 목표

`build_mt_packet()`가 INVITE 외 메서드도 최소한 **의미 있는 기본 payload**를 만들도록 보강했다.

### 변경 파일

- `src/volte_mutation_fuzzer/generator/mt_packet.py`

### 추가된 기본 동작

#### 3-1. INFO

- 기본 `Info-Package: dtmf`
- 기본 body:
  - `Content-Type: application/dtmf-relay`
  - `Signal=<seed % 10>`
  - `Duration=160`

#### 3-2. PUBLISH

- 기본 `Event: presence`
- 기본 `Expires: 3600`
- 기본 body:
  - `Content-Type: application/pidf+xml`
  - PIDF XML body 생성

#### 3-3. NOTIFY

- 기본 `Event: presence`
- 기본 `Subscription-State: active;expires=3600`
- 기본 body:
  - `Content-Type: application/pidf+xml`
  - PIDF XML body 생성

추가로:

- `event_package="refer"`를 넘기면
  - `Content-Type: message/sipfrag;version=2.0`
  - `SIP/2.0 200 OK` body 생성

#### 3-4. UPDATE

- 기본 `Session-Expires: 1800`
- 기본 `Min-SE: 90`
- 기본 body:
  - `Content-Type: application/sdp`
  - seed를 반영한 SDP body 생성

#### 3-5. SUBSCRIBE

- 기존 `Event: presence`, `Expires: 3600` 유지
- 이제 `event_package` override 가능
- 기본 body는 여전히 없음

### 현재 연결 범위 메모

`build_mt_packet()` 자체는 `event_package` / `info_package` 인자를 받도록 확장했지만,
현재 `campaign/core.py` 와 `sender/cli.py` 호출 경로에서는 이 값을 외부에서 전달하지 않는다.

따라서 현재 실사용 기본값은 다음과 같다:

- `INFO` → `dtmf`
- `SUBSCRIBE / PUBLISH / NOTIFY` → `presence`

즉 이번 변경은 **wire-text builder 능력 확장**까지 완료했고,
이를 사용자가 선택적으로 제어하는 CLI/config 연결은 후속 작업으로 남겨두었다.

### 구현 메모

가능한 한 기존 `sip.bodies` 구현을 재사용했다:

- `DtmfRelayBody`
- `PIdfBody`
- `SipfragBody`
- `SDPBody`

즉 MT wire-text 경로를 새로 따로 발명하기보다,
이미 존재하는 body 모델의 의미를 MT 패킷에도 반영하는 방향으로 맞췄다.

## 4. MT packet 직접 테스트 추가

### 변경 파일

- `tests/generator/test_mt_packet.py` (신규)

### 추가한 테스트

1. `MESSAGE` body가 seed별로 달라지고 같은 seed에서 재현되는지
2. `MESSAGE` explicit `body=` override가 유지되는지
3. `SUBSCRIBE`가 `Event: presence`, `Expires: 3600`, empty body를 갖는지
4. `OPTIONS`가 body 없이 `Content-Length: 0`인지
5. `INFO`가 기본 DTMF body/header를 갖는지
6. `PUBLISH`가 기본 presence PIDF body/header를 갖는지
7. `NOTIFY`가 기본 presence PIDF body/header를 갖는지
8. `NOTIFY event_package="refer"`가 sipfrag body를 갖는지
9. `UPDATE`가 SDP body + session timer header를 갖는지

이전에는 `mt_packet` 경로가 대부분 **간접 커버리지**에 가까웠는데,
이제는 MT wire-text 생성 자체를 직접 회귀 테스트로 고정했다.
